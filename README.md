# S3 Folder Watcher — Timeweb Cloud

Фоновая служба Windows, которая автоматически отслеживает указанную папку и загружает новые/изменённые файлы в S3-хранилище Timeweb Cloud.

**Python не требуется** — скачайте готовый EXE из [Releases](../../releases) и запустите.

---

## Возможности

- **Мониторинг в реальном времени** — отслеживает создание, изменение и перемещение файлов
- **Рекурсивное наблюдение** — следит за вложенными папками, сохраняя структуру в S3
- **Защита от дублей** — хеширует файлы (MD5) и не загружает повторно
- **Debounce** — ждёт завершения записи файла перед загрузкой
- **Multipart-загрузка** — автоматически для больших файлов (> 50 МБ)
- **Автоповтор** — при ошибке повторяет загрузку (настраиваемое число попыток)
- **Фильтрация** — по расширениям файлов и паттернам игнорирования
- **Автоустановка службы** — при запуске EXE автоматически регистрирует Windows-службу
- **Автозапуск** — служба стартует вместе с Windows
- **Автоперезапуск при сбоях** — служба перезапускается автоматически через 10 секунд
- **Периодическое сканирование** — подстраховка на случай пропуска событий файловой системы
- **Логирование** — все действия записываются в `s3_watcher.log`

## Быстрый старт (EXE)

### 1. Скачайте

Перейдите в [Releases](../../releases) и скачайте:
- `S3FolderWatcher.exe`
- `config.example.json`

Переименуйте `config.example.json` в `config.json` и положите оба файла в одну папку.

### 2. Настройте подключение

Откройте `config.json` в любом текстовом редакторе:

```json
{
    "s3_access_key": "ваш Access Key из панели Timeweb",
    "s3_secret_key": "ваш Secret Key из панели Timeweb",
    "s3_bucket": "имя-вашего-бакета",
    "watch_folder": "C:\\Путь\\К\\Папке"
}
```

**Где взять ключи:**
1. Войдите в [панель управления Timeweb Cloud](https://timeweb.cloud/my/storage)
2. Выберите нужный бакет -> вкладка "Дашборд"
3. Скопируйте Access Key, Secret Key и имя бакета

### 3. Запустите

Дважды кликните по `S3FolderWatcher.exe`. Программа автоматически:

1. Запросит права администратора (UAC)
2. Установит себя как Windows-службу
3. Включит автозапуск при старте Windows
4. Настроит автоперезапуск при сбоях
5. Сразу запустит службу

**Готово!** Файлы, помещённые в папку наблюдения, будут загружаться в S3.

## Управление службой

### Режимы запуска

| Команда | Действие |
|---------|----------|
| `S3FolderWatcher.exe` | Автоустановка + запуск службы |
| `S3FolderWatcher.exe --console` | Консольный режим (для отладки) |

### Команды управления

Выполняются в командной строке от имени администратора:

| Команда | Действие |
|---------|----------|
| `sc start S3FolderWatcher` | Запустить службу |
| `sc stop S3FolderWatcher` | Остановить службу |
| `sc delete S3FolderWatcher` | Удалить службу |
| `sc query S3FolderWatcher` | Проверить статус |

Служба также видна в `services.msc` (Службы Windows) под именем **"S3 Folder Watcher (Timeweb Cloud)"**.

## Настройки config.json

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `s3_endpoint` | URL S3-хранилища | `https://s3.twcstorage.ru` |
| `s3_region` | Регион | `ru-1` |
| `s3_access_key` | Ключ доступа | -- |
| `s3_secret_key` | Секретный ключ | -- |
| `s3_bucket` | Имя бакета | -- |
| `s3_prefix` | Префикс (подпапка) в бакете | `""` (корень) |
| `watch_folder` | Папка для наблюдения | `C:\WatchFolder` |
| `scan_interval_sec` | Интервал периодического сканирования (сек, 0 = выкл) | `60` |
| `upload_existing_on_start` | Загрузить существующие файлы при старте | `false` |
| `file_extensions` | Фильтр расширений (пусто = все файлы) | `[]` |
| `ignore_patterns` | Игнорировать файлы с этими паттернами | `.tmp`, `~$` и др. |
| `max_retries` | Число повторных попыток загрузки | `3` |
| `retry_delay_sec` | Задержка между попытками (сек) | `5` |
| `multipart_threshold_mb` | Порог multipart-загрузки (МБ) | `50` |
| `log_level` | Уровень логов: `DEBUG`, `INFO`, `WARNING` | `INFO` |

S3-ключи также можно задать через переменные окружения `S3_ACCESS_KEY` и `S3_SECRET_KEY` — они имеют приоритет над config.json.

### Примеры фильтров

Загружать только бэкапы 1С:
```json
"file_extensions": [".dt", ".cf", ".bak", ".zip", ".7z"]
```

Загружать только изображения:
```json
"file_extensions": [".jpg", ".jpeg", ".png", ".gif", ".webp"]
```

Загружать только документы:
```json
"file_extensions": [".pdf", ".docx", ".xlsx"]
```

## Решение проблем

**Не подключается к S3:**
- Проверьте правильность `s3_access_key` и `s3_secret_key`
- Убедитесь, что бакет существует в панели Timeweb Cloud
- Проверьте интернет-соединение

**Служба не запускается:**
- Убедитесь, что `config.json` лежит рядом с `S3FolderWatcher.exe`
- Проверьте логи в файле `s3_watcher.log`
- Проверьте статус: `sc query S3FolderWatcher`

**Файлы не загружаются:**
- Проверьте, что файл не попадает под `ignore_patterns`
- Если задан `file_extensions` — убедитесь, что расширение файла в списке
- Установите `"log_level": "DEBUG"` для подробных логов
- Убедитесь, что файл не заблокирован другой программой
