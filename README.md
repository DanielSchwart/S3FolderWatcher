# S3 Folder Watcher — Timeweb Cloud

Фоновая служба Windows, которая автоматически отслеживает папку и загружает
новые/изменённые файлы в S3-хранилище Timeweb Cloud.

---

## Возможности

- **Мониторинг в реальном времени** — отслеживает создание, изменение и перемещение файлов
- **Рекурсивное наблюдение** — следит за вложенными папками, сохраняя структуру в S3
- **Защита от дублей** — хеширует файлы (MD5) и не загружает повторно
- **Debounce** — ждёт завершения записи файла перед загрузкой
- **Multipart-загрузка** — автоматически для больших файлов (по умолчанию > 50 МБ)
- **Автоповтор** — при ошибке повторяет загрузку (настраиваемое число попыток)
- **Фильтрация** — по расширениям и паттернам игнорирования
- **Два режима** — консольный (для тестирования) и Windows-служба (для продакшена)
- **Логирование** — все действия записываются в `s3_watcher.log`

## Быстрый старт

### 1. Установите Python

Скачайте Python 3.9+ с [python.org](https://www.python.org/downloads/).
При установке обязательно отметьте **"Add Python to PATH"**.

### 2. Скачайте файлы программы

Поместите все файлы в одну папку, например `C:\S3Watcher\`.

### 3. Установите зависимости

Откройте командную строку (cmd) и выполните:

```
cd C:\S3Watcher
pip install -r requirements.txt
```

### 4. Настройте подключение

Откройте `config.json` в текстовом редакторе и заполните:

```json
{
    "s3_access_key": "ваш Access Key из панели Timeweb",
    "s3_secret_key": "ваш Secret Key из панели Timeweb",
    "s3_bucket": "имя-вашего-бакета",
    "watch_folder": "C:\\Путь\\К\\Папке"
}
```

**Где взять ключи:**
1. Войдите в [панель управления Timeweb Cloud](https://timeweb.cloud/my/storage)
2. Выберите нужный бакет → вкладка «Дашборд»
3. Скопируйте Access Key, Secret Key и имя бакета

### 5. Протестируйте в консоли

```
run_console.bat
```

или

```
python s3_folder_watcher.py
```

Положите файл в наблюдаемую папку — он должен загрузиться в S3.

### 6. Установите как Windows-службу

Запустите **от имени Администратора**:

```
install_service.bat
```

Или вручную:

```
python s3_folder_watcher.py install
net start S3FolderWatcher
```

## Управление службой

| Команда | Действие |
|---------|----------|
| `net start S3FolderWatcher` | Запустить службу |
| `net stop S3FolderWatcher` | Остановить службу |
| `python s3_folder_watcher.py remove` | Удалить службу |
| `python s3_folder_watcher.py update` | Обновить службу после изменений |

Служба также видна в `services.msc` (Службы Windows).

## Настройки config.json

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `s3_endpoint` | URL S3-хранилища | `https://s3.twcstorage.ru` |
| `s3_region` | Регион | `ru-1` |
| `s3_access_key` | Ключ доступа | — |
| `s3_secret_key` | Секретный ключ | — |
| `s3_bucket` | Имя бакета | — |
| `s3_prefix` | Префикс (подпапка) в бакете | `""` |
| `watch_folder` | Папка для наблюдения | `C:\WatchFolder` |
| `upload_existing_on_start` | Загрузить существующие файлы при старте | `false` |
| `file_extensions` | Фильтр расширений (пусто = все) | `[]` |
| `ignore_patterns` | Игнорировать файлы с этими паттернами | `.tmp`, `~$` и др. |
| `max_retries` | Число повторных попыток | `3` |
| `multipart_threshold_mb` | Порог multipart-загрузки (МБ) | `50` |
| `log_level` | Уровень логов: DEBUG, INFO, WARNING | `INFO` |

### Примеры настройки фильтров

Загружать только изображения:
```json
"file_extensions": [".jpg", ".jpeg", ".png", ".gif", ".webp"]
```

Загружать только документы:
```json
"file_extensions": [".pdf", ".docx", ".xlsx"]
```

## Решение проблем

**Не подключается к S3:**
- Проверьте правильность `s3_access_key` и `s3_secret_key`
- Убедитесь, что бакет существует
- Проверьте интернет-соединение

**Служба не запускается:**
- Убедитесь, что установлен `pywin32` (`pip install pywin32`)
- Запускайте `install_service.bat` от имени Администратора
- Проверьте логи в `s3_watcher.log`

**Файлы не загружаются:**
- Проверьте, что файл не в списке игнорирования
- Установите `"log_level": "DEBUG"` для подробных логов
- Убедитесь, что файл не заблокирован другой программой
